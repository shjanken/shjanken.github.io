`iptables`命令的语法：
---

        iptables [-t Table (规则写到那张表上)] COMMAND CHAIN {匹配标准} -j {处理办法}

命令（`COMMD`）
----

`-A` ： 添加一条规则

`-I chain [rulenumber]` ： 插入一条规则

`-D chain rulenumber` : 删除一条规则

`-R chain rulenumber` : 修改一条规则



匹配标准
----

- 通用匹配：

    `-s` 或 `--src`：表示匹配源地址 

     `-d` 或 `--dst`： 表示匹配目标地址

    `-p {tcp|udp|imcp}`： 匹配数据包的协议

    `-i interface`： 制定数据报文的流入接口（网卡）

    `-o` interface`： 指定数据报文的流出接口（网卡）

    `sport port`: 匹配源端口

    `dport port`: 匹配目标端口


处理办法: `-j`
----

`ACCEPT` 接受报文
    
`DROP` 丢弃报文，不通知对方
    
`REJECT` 丢弃报文，并通知对方拒绝信息。
      

-----

示例：
-----
    
- 添加允许 `ssh` 访问

        iptables -A INPUT -p tcp --dport 22 -j ACCEPT


- 允许 `ssh` 响应。
    
        iptables -A OUTPUT -p tcp --sport 22 -j ACCEPT
        
    *如果需要将服务器的安全等级设置的非常高（ 将所有的 `input` 报文和 `output` 报文都设置为默认不允许），则在设置默认规则规则之前，一定要添加以上两条 `ssh` 规则。不然一旦设定了默认规则。 将导致 `ssh` 无法连接*


- 允许 `icmp` 报文（`ping`）
    
        iptables -A OUTPUT -p icmp -j ACCEPT

        iptables -A INPUT -p icmp -j ACCEPT
        


---

补充： 
---

* 能否使用自定义链？ 
   可以使用自定义链，但只有在被默认链中调用才会有作用。如果自定义链中没有被匹配成功，还需要有返回机制返回到默认链中。 

* 每个规则都有2个内置的计数器。一个记录了被匹配的报文个数。一个记录了被匹配的报文总大小。
